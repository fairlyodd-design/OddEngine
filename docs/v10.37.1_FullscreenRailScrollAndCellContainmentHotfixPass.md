# v10.37.1_FullscreenRailScrollAndCellContainmentHotfixPass

Purpose: fix the issues reported after v10.37.0:

- left sidebar does not scroll in fullscreen mode
- right activity/assistant rail does not scroll in fullscreen mode
- cells/cards on different panels smash together or run behind other areas
- some content becomes unreachable/no way to move it

This is a surgical CSS containment pass. It does not touch Trading logic, CardGODMode logic, routing, panel data, or panel ownership.

## Files

- `ODUI_FULLSCREEN_RAIL_AND_CELL_CONTAINMENT.css`
- `scripts/apply_v10.37.1_FullscreenRailScrollAndCellContainmentHotfixPass.ps1`
- `RUN_APPLY_ODUI_FULLSCREEN_FIX.bat`

## How to apply

Unzip into `C:\OddEngine`, then run:

```powershell
.\RUN_APPLY_ODUI_FULLSCREEN_FIX.bat
npm --prefix ui run build
npm run dev:desktop
```

## What changed

- Makes `html`, `body`, `#root`, `.layout`, `.layoutWide`, `.rail`, `.main`, and `.activityRail` participate in a true fullscreen scroll model.
- Left rail and right rail get their own vertical scroll lanes.
- Main content gets its own vertical scroll lane.
- Grid/flex children get `min-width:0` and `max-width:100%` so they stop forcing columns wider than the shell.
- Nav text truncates instead of pushing behind the pin button.
- Tables/code/textareas/images get containment rules so they do not blow out cards.
- Responsive breakpoints preserve mobile/tablet behavior.

## Acceptance checks

1. Fullscreen app: left sidebar scrolls.
2. Fullscreen app: right activity rail scrolls when visible.
3. Home, Homie, Builder, Dev Engine, Money, Family Budget: cells no longer run behind neighboring cards.
4. Trading still loads and CardGODMode still works.
5. `npm --prefix ui run build` passes.
