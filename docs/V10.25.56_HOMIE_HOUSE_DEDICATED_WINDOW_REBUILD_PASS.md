# v10.25.56 — Homie House Dedicated Window Rebuild Pass

## Goal
End the ongoing companion-window patch loop by fixing the routing architecture.

## Root issue
The detached Homie window was still opening the standard forced-panel shell. Even when the Coinstore panel internally tried to render companion-only mode, App.tsx was still wrapping that panel with shell summary, command bar, dock, and other desk chrome.

## Fix
This pass adds a dedicated `companionOnlyWindow` route in `ui/src/App.tsx` based on `phoenixCompanion=1`. When that query flag is present, App renders only the target panel content and skips the normal forced-panel shell. The Coinstore Phoenix panel already has a companion-only branch, so that branch can now take over cleanly.

## Outcome
- one Homie, not two
- lower-right Homie = the source of truth
- expanded House in-panel still works
- detached House opens as its own real window through desktop `openWindow`
- detached House shows only Homie's UI, not the duplicated trading shell
