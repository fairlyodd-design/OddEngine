# v10.38.12c_HomieVoiceSnapshotSpeakingTypecheckFixPass

## Fix

Repairs CI typecheck failure:

```txt
src/panels/Homie.tsx(...): error TS2339:
Property 'speaking' does not exist on type 'VoiceEngineSnapshot'.
```

## Cause

Recent Homie warmth/presence logic read `snapshot.speaking` directly, but `VoiceEngineSnapshot` does not declare that property.

## What this does

- Adds a type-safe helper:

```ts
isHomieVoiceSnapshotSpeaking(snapshot: VoiceEngineSnapshot)
```

- Checks optional runtime fields through a widened local type:
  - `speaking`
  - `isSpeaking`
  - `ttsPlaying`
  - text status/mode hints
- Replaces direct `.speaking` access in `Homie.tsx`.

## Safety

- Typecheck-only hotfix.
- No backend rewrite.
- No voice engine internals changed.
- No Trading changes.
- No CardGODMode changes.
- No shell changes.
- Visual Homie work preserved.
