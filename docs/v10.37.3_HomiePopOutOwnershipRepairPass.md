# v10.37.3_HomiePopOutOwnershipRepairPass

## Why this exists
The OD UI rail containment passes fixed/sidebar scroll pressure, but Homie Buddy was accidentally caught by global containment rules. The floating launcher became squeezed into a narrow vertical column and overlapped right-rail action content.

## What this fixes
- Restores Homie pop-out/launcher ownership.
- Forces the launcher to remain a fixed 90x90 round button.
- Prevents vertical text/writing-mode inheritance inside Homie.
- Gives the open Homie panel a stable width and independent scroll.
- Adds right-rail bottom breathing room so Homie does not cover final action buttons.
- Keeps buttons horizontal and readable.

## What it does not touch
- Trading logic
- CardGODMode logic
- Panel routing
- Voice logic
- Homie memory/reply logic

## Apply
From `C:\OddEngine`:

```powershell
.\RUN_APPLY_ODUI_HOMIE_POP_OUT_REPAIR.bat
npm --prefix ui run build
npm run dev:desktop
```
