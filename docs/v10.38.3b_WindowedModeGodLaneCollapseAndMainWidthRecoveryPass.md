# v10.38.3b_WindowedModeGodLaneCollapseAndMainWidthRecoveryPass

## Mission

Fix the remaining windowed-mode squish where GodMode/side-card lanes stay as a skinny column between the main content and assistant rail.

## What the screenshot showed

The main shell is mostly better, but a side lane still sits beside the main content. Text like Market Map / Time Machine / Family Budget wraps vertically because the lane is too narrow.

## Fix

CSS-only:

- PanelShell uses one main column.
- Non-main panel children drop below the panel instead of beside it.
- Known assistant/God lane/side stack classes become full-width.
- Side stacks turn into responsive horizontal card grids.
- Right rail becomes hover/peek in narrower widths.
- Main panel keeps protected readable width.
- Last-resort horizontal scroll is allowed before vertical text crush.

## Safety

No Trading logic changes.
No CardGODMode internals changes.
No Homie backend/voice/memory changes.
No routing rewrite.
No component rewrite.
