# v10.38.21b_HomieTrue3DAvatarMoodScopeFixPass

## Fix

Repairs the v10.38.21 typecheck/runtime failure:

```txt
Cannot find name 'livingPresenceState'
livingPresenceState is not defined
```

## Cause

The 3D avatar mount inserted this into `Homie.tsx`:

```tsx
<HomieTrue3DAvatar size="main" mood={livingPresenceState || "idle"} />
```

But `livingPresenceState` is not in scope in the main Homie panel.

## What this does

- Replaces the unsafe mood prop in `Homie.tsx` with:

```tsx
<HomieTrue3DAvatar size="main" />
```

- Lets `HomieTrue3DAvatar` default to idle internally.
- Applies the same defensive replacement in `HomieBuddy.tsx` if needed.

## Safety

- UI-only.
- Keeps the v10.38.21 3D viewer lane.
- Keeps PNG/CSS fallback.
- No backend rewrite.
- No voice engine internals.
- No Trading changes.
- No CardGODMode changes.
- No shell changes.
