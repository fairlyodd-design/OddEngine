# v10.38.22_HomieRiggedGLBAvatarPrototypePass

## Mission

Create the real avatar lane: stop relying on image mashups and install a prototype GLB/rig asset path.

## What this pass does

- Adds a valid `.glb` prototype asset:
  - `ui/public/homie/homie-rigged-prototype.glb`
- Adds an asset manifest:
  - `ui/public/homie/homie-rigged-prototype.manifest.json`
- Adds CSS cleanup so the old anime/composite PNG no longer bleeds through behind the true 3D viewer.
- Preserves the existing `HomieTrue3DAvatar` procedural viewer as the Phase 1 visible interactive avatar.
- Keeps the PNG/CSS avatar only as fallback, not as a visible mashup layer.

## Important truth

This is a **prototype rig lane**, not the final production character rig yet.

The GLB included here is a repo-safe starter skeleton/metadata asset for Homie:
- root
- hips
- spine
- chest
- neck
- head
- cap
- glasses
- face/mouth/eye hooks
- arms/hands
- legs/feet

It gives OddEngine a real GLB asset path and model manifest now, so the next pass can swap in a production rigged model from Blender / VRM / VRoid / Ready Player Me-style / custom GLB pipeline.
