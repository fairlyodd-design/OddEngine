# v10.37.0_ODUIStabilityAndCleanupAuditPass

## Diagnosis

The biggest UI debt is not one broken panel. It is shared rhythm drift:

1. Panel headers are implemented with local inline layout patterns, so Homie, Home, Dev Engine, Builder, Money, and Family Budget do not feel like the same OS.
2. Cards have heavy gradients, inconsistent padding, and duplicated shadows, making panels feel louder and more fragile than they are.
3. Shared controls such as badges, chips, tabs, and card headers lack one final authority layer.
4. CardGODMode and widget controls are mostly working, but the visual/control stacking needs calmer z-index discipline so controls do not feel like they are fighting the panel.
5. Large tables/forms can stretch panel content and make pages feel stitched together.

## Strategy

This pass is surgical:

- Do not touch Trading internals.
- Do not rewrite CardGODMode.
- Do not change routing or panel ownership.
- Replace only two shared components with backwards-compatible full drop-ins:
  - `ui/src/components/PanelHeader.tsx`
  - `ui/src/components/CardFrame.tsx`
- Append one scoped OD UI CSS block to `ui/src/styles.css`.
- Update the UI version label to `10.37.0`.

## What changed

- One shared panel header class system: `oduiPanelHeader`, `oduiPanelTitle`, `oduiPanelHeaderActions`.
- One shared card frame rhythm: `oduiCardFrame`, `oduiWidgetHeader`, `oduiWidgetBody`.
- Calmer global card surface, spacing, badge, tab, and form defaults.
- Safer MutationObserver handling in `CardFrame` using `requestAnimationFrame` so class churn does not trigger extra render jitter.
- Z-index discipline for God controls without changing behavior.
- Responsive behavior for cramped headers.
