# v10.25.55a — Homie Companion Separate Window Hotfix

## Why this pass exists
The earlier v10.25.55 native bridge route added debugging confusion and a fragile patch path.
This hotfix simplifies the second-screen companion behavior.

## What changed
- Detached Homie now uses `window.__ODD__.openWindow()` when desktop mode is available.
- The detached companion opens as a real separate OddEngine window with `panel=CoinstoreBTCUSDTFutures&phoenixCompanion=1`.
- The main Phoenix panel syncs companion state through localStorage and BroadcastChannel.
- The detached window renders a focused companion-only surface using the existing Homie memoji.
- A repair script removes the unsafe standalone `mainWindow.webContents.openDevTools({ mode: "detach" });` line if it was added manually.

## Result
- no custom Electron main/preload patch required for second-screen Homie
- no renderer popup trap when desktop openWindow exists
- cleaner, more reliable second-monitor behavior
