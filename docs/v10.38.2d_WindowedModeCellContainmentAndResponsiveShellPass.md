# v10.38.2d_WindowedModeCellContainmentAndResponsiveShellPass

## Mission

Fix windowed-mode squished cells/cards while preserving fullscreen behavior.

## What the video showed

In windowed mode the app keeps three shell columns plus panel assistant/card lanes active. The main area gets squeezed, causing cards, side cards, and labels to collapse into thin vertical columns. Fullscreen works because there is enough width.

## Fix

CSS-only responsive shell containment:

- Responsive left rail / right rail widths.
- Main panel is `minmax(0, 1fr)`.
- Assistant dock drops under the panel in medium windows.
- Right activity rail becomes edge-peek/hover in narrower desktop windows.
- Small windows become a single-lane layout.
- Cards, tables, inputs, code, media, and buttons wrap instead of clipping.
- Prevents vertical one-letter text columns.
- Keeps fullscreen roomy.

## Safety

No Trading logic changes.
No CardGODMode internals changes.
No Homie backend/voice/memory changes.
No routing rewrite.
No component rewrite.

## Apply

```powershell
cd C:\OddEngine
.\RUN_APPLY_WINDOWED_CELL_CONTAINMENT.bat
npm --prefix ui run build
npm run dev:desktop
```

## Verify

- Fullscreen still looks normal.
- Windowed mode no longer crushes cells into vertical columns.
- Right rail becomes a peek rail at narrower widths.
- Assistant dock drops under panel instead of squeezing it.
- Buttons/action rows wrap.
