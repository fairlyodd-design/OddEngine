# v10.37.2_TrueRailScrollAndActionContainmentPass

## Target
Fix the fullscreen shell layout issue where the left/right rails do not scroll and action buttons/cards get clipped or pushed behind nearby UI.

## What this pass changes
- Adds a shell-level CSS containment block to `ui/src/styles.css`.
- Forces left rail, main lane, and right activity rail to be independent scroll lanes.
- Adds bottom padding to the right activity rail so Homie Buddy does not cover action cards.
- Prevents cards/grid children from exceeding their columns.
- Makes action rows wrap instead of cutting off buttons.
- Keeps scope surgical: no Trading logic, CardGODMode logic, routes, or panel data flow changes.

## Apply
From `C:\OddEngine`:

```powershell
.\RUN_APPLY_ODUI_RAIL_SCROLL_FIX.bat
npm --prefix ui run build
npm run dev:desktop
```

## Verify
1. Fullscreen the Electron app.
2. Use the mouse wheel over the left sidebar: it should scroll independently.
3. Use the mouse wheel over the right Mission Control Rail: it should scroll independently.
4. Open Trading / Options SaaS / Brain / Money and inspect bottom action buttons.
5. Confirm no cards are trapped behind adjacent panels and no buttons are chopped off.
